## Summary
Implements scope **216A** on top of the current branch by extending our `gemma4_audio` runtime from audio/text-only into a practical image-capable multimodal path, then wiring the capability through backend contracts and voice/cockpit runtime UX.

This PR includes:
1. runtime/model inventory hardening for `gemma4_audio` based on real local HF snapshots,
2. daemon/API support for image input (`url`, `data`, multipart files) and image capability metadata,
3. frontend runtime controls for image workflows (including drag-and-drop and disk file picker),
4. sidebar/runtime ordering and visibility fixes in voice screen.

## Backend changes
### `services/gemma4_audio_runtime/*`
- `engine.py`
  - `respond()` now supports optional image list + text-only/image-only flows (audio optional).
  - adds image token budget in daemon params/status.
  - marks runtime status with `supports_image_input`.
- `schemas.py`
  - `ContentItem` now supports `type: "image"` with `url|data|path`.
  - `Capabilities` extended with image fields.
  - daemon config/status extended with `image_token_budget` and `supports_image_input`.
- `main.py`
  - `/v1/respond` accepts image via JSON and multipart (`image`, `images[]`).
  - OpenAI-compatible `/v1/chat/completions` supports `image_url` content parts.
  - runtime status and models endpoints expose image capability.

### `venom_core/*`
- `venom_core/services/gemma4_audio_models.py`
  - new helper to resolve available target/assistant models from real local snapshots.
- `venom_core/api/routes/system_llm.py`
  - capability contract for `gemma4_audio` now includes image support + image budget options.
- `venom_core/api/audio_stream.py`
  - runtime snapshot includes image support + image token budget.
- `venom_core/config.py`
  - adds `GEMMA4_AUDIO_IMAGE_TOKEN_BUDGET`.
