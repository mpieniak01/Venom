# Lista podstron do migracji na nowy layout VENOM

| Strona (template) | Skrypt JS | Główna funkcja | Status migracji |
|-------------------|-----------|----------------|------------------|
| `index.html` (Cockpit) | `web/static/js/app.js` | Zarządzanie kolejką, chat, telemetry | ✅ ukończony – layout `_szablon.html`, nowy menedżer zakładek (ARIA + klawiatura) w `app.js`, lewa kolumna (telemetria/zakładki) oraz prawa kolumna (governance + chat) 1:1 z legacy, wszystkie ID JS zachowane, status banner i moduły (jobs/memory/models/history/voice) działają na komponentach Venom **+** preferencje użytkownika (aktywna zakładka, filtry i auto-refresh Historia/Zadania/Pamięć oraz domyślna zakładka Cockpitu) są zapisywane w `localStorage`, a zakładki Historia, Zadania, Pamięć i Modele mają panele „Insights” (statystyki żądań, jobów, lekcji/grafu oraz inwentarza modeli) oraz szybkie przyciski „Resetuj filtry” umożliwiające natychmiastowe przywrócenie ustawień domyślnych bez ręcznego cofania zmian. Dodano modal „Centrum preferencji” z podsumowaniem zapisanych stanów i akcjami czyszczenia preferencji Cockpitu lub zewnętrznych modułów (Flow, Inspector, War Room, Brain). |
| `brain.html` (The Brain) | `web/static/js/brain.js` | Graf wiedzy Cytoscape, filtry | ✅ ukończony – layout Venom + panel sterowania, zapamiętywanie filtrów/szukania/auto-refresh (localStorage) oraz eksport grafu (JSON/PNG) bezpośrednio z UI; header `venom-page` pokazuje live licznik węzłów/krawędzi, status i timestamp (zasilane przez `brain.js`), a sekcja „Brain Insights” informuje ile elementów jest widocznych po filtrach i wyszukiwaniu |
| `flow_inspector.html` (Flow Inspector) | `web/static/js/flow_inspector.js` | Podgląd przepływów zadań | ✅ layout Venom + powłoka `venom-page`, panel sterowania (wyszukiwarka/filtry/auto-refresh), sekwencje Mermaid z możliwością kopiowania kodu i pobrania PNG/SVG prosto z UI; nagłówek pokazuje liczbę aktywnych zadań i ostatnie odświeżenie (JS) **+** preferencje operatora (fraza, filtr, auto-refresh) zapisywane są w `localStorage`, więc widok odtwarza poprzedni kontekst pracy |
| `inspector.html` (Inspector) | `web/static/js/inspector.js` | Szczegóły zadań/logów | ✅ ukończony – layout Venom + powłoka `venom-page`, panel sterowania bez Alpine (vanilla `InspectorApp`, filtry, auto-refresh, zoom), eksport śladu (kopiuj Mermaid, pobierz PNG/SVG, export JSON) **+** przypinanie śladów (localStorage, sekcja chipsów) i eksport pojedynczego kroku jako JSON; header pokazuje liczbę widocznych śladów/all oraz liczbę pinów, a ustawienia listy (fraza, filtr, auto-refresh) są zapamiętywane w `localStorage` |
| `strategy.html` (War Room) | `web/static/js/strategy.js` | Strategia / planowanie | ✅ nowy layout Venom + odświeżone renderowanie JS; header `venom-page` pokazuje status roadmapy (postęp %, milestones/tasks, health) i auto-refresh, a preferencja auto-refresh jest zapisywana w `localStorage` |
| `index_.html` (legacy UI) | n/a | Stare demo | ✅ usunięty po wdrożeniu nowego layoutu |

> `_navbar.html` (global) – ✅ Venom Global Nav z neonowym menu i kontrolkami (Autonomia, Tryb Eco) jest wspólną belką dla wszystkich widoków poza Cockpitem, dzięki czemu migracje z tej listy korzystają ze spójnego headera.
> Po zakończeniu prac na `index.html` należy przenieść ten sam header/sidebar, design tokens i konwencje CSS do pozostałych stron. Każda z nich ma swój JS – trzeba zachować kompatybilne ID.
